React (completed)

React, ein voll individuell zusammenstellbares Framework. So multi-funktionell, wie auch einzigartig. Nicht nur im Bereich des Web-Developments gut einsetzbar, sondern auch im Bereich Web Progressiv Webapps. In Kombination mit Redux für kleine und große Vorhaben bestens geeignet. Als SPA oder Multipage oder auch nur teilweise in einer statischen HTML-Seite einsetzbar.

Redux (completed)

Redux ist ein sehr gutes Framework in Kombination mit React um verschiedene States einfacher zu handhaben. Es macht den Code um einiges leaner und besser verständlich.

Abschluss des Backend-Parts des Main-Stacks

Node.JS (completed)

Eine Sache warum ich JavaScript so interessant finde, ist z.B. Node.JS. Durch Node.JS lässt sich Javascript auch außerhalb des Browsers ausführen, womit die Möglichkeiten unbegrenzt sind. Durch Node.JS ist Javascript nicht mehr "nur" die Sprache die einzelne Events im Browser verarbeitet, sondern auch zu I/O Prozessen genutzt werden kann, womit Javascript auch z.B. Zugriff auf das Filesystem ermöglicht wird. Bei Node.JS beherrsche ich die Basics.

Express.JS (completed)

Wer mit Node.js arbeitet, kennt sicherlich auch Express.JS. Ein Framework was die Arbeit mit Node.js ungemein erleichtert. Mir derzeit nur bekannt, im Rahmen eines Webservers. Von simplen Routing bis JSON Daten parsen sind mir die Vorgehensweisen bekannt.

MongoDB (completed)

Wer Daten strukturiert und sicher aufbewahren möchte, kommt um eine gute Datenbank nicht drum herum. Ebenfalls auf JavaScript aufbauend, befasste ich mich mit MongoDB. In Kombination mit mit Mongoose ein starkes Duo.

UI/UX (completed)

Userinterface und Userexperience, 2 interessante Themen. Dafür absolvierte ich einen Kurs, um das best mögliche raus zu holen. Mir sind Strategien bekannt, sowie die Design Patter und auch die Entwicklung. Für die Entwicklung von Wireframes nutze ich am liebsten JustInMind.

Next.JS

Warum will ich den Kurs belegen?
Next.JS kombiniert Front-End mit Back-End und das auf Basis von React. Da mein Stack auf React basiert, empfinde ich es quasi als must have. Aber nicht nur das, die integrierte Opimierung von Bilder, SEO und viel mehr, machen das ganze Paket richtig spannend.

Angular

Warum will ich den Kurs belegen?
Angular, eines der 3 verbreitesten Frameworks, neben React und Vue, für Javascript. Da Angular auf Typescript basiert, ist es eine hervorragende Möglichkeit, meine Typescript Kenntnisse zu vertiefen. Dazu lerne ich noch einen weiteres Framework und zeitgleich erweitere ich meinen Stack noch um den MEAN Stack. Das ist aber noch nicht alles. Mit erlangen der Kenntnisse über Angular, bin ich auch in der Lage, zwischen 2 großen Frameworks, das optimalste Framework für das Projekt zu wählen.

Three.js

Warum will ich den Kurs belegen?
Three.js ist ein grafisch-faszinierendes Framework, meiner Meinung nach. Damit lassen sich nicht nur 3D Games erstellen, sondern auch faszinierende Effekte. Mit diesem Framework erhoffe ich mir, den Webseiten ein einmaliges und einzigartiges Ausssehen verleihen zu können.

Clean Code (completed)

Clean Code, macht Spaß. Clean Code macht nicht nur dem Author Spaß, sondern auch dem Leser und jedem der mit diesem Code arbeiten muss. Clean Code macht, die Wartung des Codes einfacher. Es gibt so viele Vorteile durch Clean Code, dass es mir schwer fällt, alles in kürze hier zu formulieren.

Progressive Web App (completed)

Diesen Kurs habe ich belegt, da ich dieses Feature hinsichtlich der Möglichkeiten sehr interessant finde. Es gibt einem die Möglichkeit, eine App zu entwickeln, die sich anfühlt wie eine echte App und das auf Basis simplem HTML/CSS/JS. Auch die Möglichkeit, Pushmessages senden zu können und damit den Besucher immer wieder auf die Seite zu ziehen ist optimal. Aber nicht nur das, damit lässt sich auch gewisse Hardware des Smartphones nutzen, wie Kamera/Mikrofon oder spezielles Vibrationsmuster und vieles mehr.

Testing ist im allgemeinen ein wichtiges Thema. Testing zeigt Änderungen auf, die ggf. zu einem break Code führen. Im Rahmen dieses Kurses wird nicht nur Unit und Integrationtests gelehrt, sondern auch TestDriven Developement. Für guten, Bugfreien Code.

Unit & Integrationtesting mit Typescript

Dieser Kurs soll noch einmal als Auffrischung dienen, aber auch tiefer in das Thema testing mit Typescript führen. Somit kenne ich jede Möglichkeit des Testings in meiner Main Sprache.
